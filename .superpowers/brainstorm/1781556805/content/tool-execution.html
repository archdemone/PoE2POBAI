Step 3 of 5: Tool Execution

The browser needs to run MCP tools. But tools live on the server side.

A

HTTP API endpoints

Add REST endpoints for ALL 8 MCP tools to pobai-server. Browser calls them directly (e.g. GET /api/build/:id/defenses). Clean REST contract, debuggable, no WS tool execution channel needed.

B

WebSocket RPC

Browser sends tool execution requests over the existing WS: {type: "tool_call", tool: "get_defenses", args: {...}}. Server executes and sends {type: "tool_result", ...} back. Single connection, no new routes.

C

Keep /api/chat, stream via WS

Server still executes tools (like today). The change is: WS streams progress events so the browser shows each step in real-time. Frontend "drives" visually but server does the work. Simplest migration from current code.
