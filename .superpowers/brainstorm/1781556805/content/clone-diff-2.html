Task #3 — What's Patchable?

The "what-if" feature needs a patch model. Which build aspects can the LLM/API modify?

A

Items (swap/add/remove)

Change equipment by slot name: "swap current ring for Ming's Heart." LLM specifies slot + new item stats. Most common what-if scenario.

B

Skills & gems

Swap active skill gems, reconfigure supports, change gem levels. "What if I switch from Herald of Thunder to Herald of Ice?"

C

Passive tree nodes

Allocate/unallocate nodes. "What if you path through the Shadow area instead?" Requires tree structure awareness.

D

Bandit/ascendancy choice

Change ascendancy or bandit reward. "What if you went Deadeye instead of Pathfinder?"

E

Stat overrides (manual)

Directly override a computed stat (e.g. add +50% cold res). Useful when patch model doesn't cover a scenario. LLM must justify the override.

Constraint Question

A

XML-only patches

Apply patches by modifying the raw PoB2 XML. Full power but fragile. Works without the Lua bridge.

B

Structured patch format

LLM emits a structured JSON patch: {items: {"Ring 1": {...}}, skills: {...}}. Server validates and applies. Safer, limited to what the parser supports.

Select all patchable aspects, then choose XML-only vs structured patches.
