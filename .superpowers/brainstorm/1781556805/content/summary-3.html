Task #3: Design Summary

Snapshot Clone/Diff — Architecture

Decisions

CapabilitiesClone, Diff, What-if, MCP tools
PatchableItems, Skills, Passives, Bandit
Patch formatXML modification
Diff outputJSON + text + side-by-side UI
ProvenanceLinked (parent tracking)
Locationpob-mcp-server

New MCP Tools

clone_buildCopy snapshot, optional label, returns new ID + parent lineage
diff_buildsCompare two snapshots, returns JSON diff + human summary
patch_buildApply XML patch to clone, re-parse, return new snapshot
get_build_lineageTrace clone tree from root to variants

Flow: What-If

1. LLM analyzes build, proposes changes
2. LLM emits XML patch (modded items/skills/passives)
3. clone_build(snapshot_id, label="What-if: Ming's swap")
4. patch_build(clone_id, xml_patch)
5. diff_builds(original_id, clone_id)
6. LLM interprets diff, gives recommendation

Files Changed

index.tsRegister 4 new MCP tools
src/New: diff-engine.ts, patch-applier.ts

pobai-web/components/DiffView side-by-side component

Tests

Clone preserves all data + links to parent
Diff detects item swaps, skill changes, passive changes
XML patch modifies builds correctly
Lineage traverses full clone tree
Self-diff returns empty (no changes)
